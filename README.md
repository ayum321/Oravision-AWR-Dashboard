# OraVision AWR Pro

Oracle AWR comparison and Root Cause Analysis dashboard.
Upload two AWR HTML reports (good baseline + bad/problem period) and get an instant,
evidence-driven diagnosis: wait event breakdown, SQL regressions, execution plan changes,
and prioritised fix recommendations — all in a single local web page.

---

## Requirements

| Requirement | Version |
|---|---|
| OS | Windows 10 / 11 (64-bit) |
| Python | 3.10 or later |
| Browser | Chrome, Edge, or Firefox |

Python download: https://www.python.org/downloads/  
**During install: tick "Add Python to PATH"**

---

## Quick Start

1. Unzip to any folder (e.g. `C:\Tools\awr-dashboard\`)
2. Double-click **`start.bat`**
3. The browser opens at `http://localhost:8000`
4. Upload one or two AWR HTML reports and click **Analyse**

> **First launch** installs Python packages (~30 seconds). All subsequent starts are instant.

To stop: press `Ctrl+C` in the terminal window that `start.bat` opened.

---

## AWR Report Formats Supported

- Oracle 11g, 12c, 19c, 21c
- AWR HTML generated by `awrrpt.sql` (standard Oracle script)
- Single report (health view) or two reports (comparison + RCA)

## Architecture

```
AWR HTML upload (browser)
  │
  ▼
FastAPI backend  (backend/main.py, port 8000)
  │
  ├── HTML parser          (services/html_parser.py)
  │     Extracts: wait events, SQL stats, load profile, system stats, I/O, redo
  │
  ├── Comparison engine    (services/comparator.py)
  │     Diffs two snapshots: elapsed Δ%, exec count Δ%, plan hash changes, new/disappeared SQL
  │
  ├── RCA engine           (services/rca_engine.py)
  │     Scores bottleneck hypotheses against 5 diagnostic dimensions:
  │       CPU saturation · wait queue · I/O · SQL regression · workload shift
  │     Assigns confidence tiers: CONFIRMED / STRONG_CANDIDATE / PROBABLE / INCONCLUSIVE
  │
  ├── Health scorer        (services/health_scorer.py)
  │     Converts raw AWR metrics into severity bands for the summary panel
  │
  ├── Recommendations      (services/recommendations.py  +  routers/recommendations.py)
  │     Maps each bottleneck class to a ranked, actionable fix list
  │
  └── Intelligence pipeline (services/awr_intelligence_v4.py)
        Enriches the RCA with pattern-matched narrative text (no LLM required)

  ▼
Jinja2 renders backend/templates/index.html
  Single-file dashboard: all JS, CSS, and HTML in one template.
  All panels are data-driven — no hardcoded mock values.
  Panels: RCA Verdict · Wait Events · SQL Comparison · Health Summary · Recommendations
```

### Key Design Decisions

| Decision | Reason |
|---|---|
| Single HTML template | Zero build pipeline; ships as one file; works offline |
| Process-local upload storage | No database dependency; keeps it a simple local tool |
| Deterministic diagnosis | No LLM dependency; reproducible results for PE audits |
| Evidence-availability metadata | Missing AWR sections reported as "unavailable", never as zero |

## Building a Release

Run from the project root:

```
python build.py
```

Options:

| Flag | Effect |
|---|---|
| *(no flags)* | Creates `oravision-awr-vYYYYMMDD.zip` in the project root |
| `--version 2.1.0` | Sets a custom version tag in the zip name |
| `--dry-run` | Prints the file manifest without writing the zip |
| `--out ./releases` | Writes the zip into a specific output folder |

The script automatically excludes all dev/audit/debug files (`_*.py`, `__pycache__`, `.env`, etc.)
so only the runtime files needed by end-users end up in the zip.

## Environment Variables (optional)

| Variable | Default | Effect |
|---|---|---|
| `ORAVISION_PORT` | `8000` | Port the server listens on |
| `ORAVISION_API_KEY` | *(unset)* | Enables `X-Api-Key` auth for hosted deployments |
| `ORAVISION_MAX_UPLOADS` | `20` | Max concurrent uploads held in memory |

Set these in a `.env` file next to `start.bat` (`.env` is excluded from release zips).
